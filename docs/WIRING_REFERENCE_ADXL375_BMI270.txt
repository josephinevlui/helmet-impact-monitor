# Head Impact Monitor: Arduino Nano 33 BLE Rev2 + ADXL375 + BMI270 (internal)
# =============================================================================
# NOTE: This is a temporary configuration while MPU-9250 headers are being
# soldered. The BMI270 is the onboard gyroscope — no wiring needed for it.
# Swap back to Revision 2 firmware once MPU-9250 headers are soldered.
# =============================================================================

==============================================================================
ADXL375 → Arduino Nano 33 BLE Rev2  (SPI) — unchanged from Rev 2
==============================================================================

  ADXL375 Pin    Nano 33 BLE Rev2 Pin    Notes
  ───────────    ─────────────────────   ──────────────────────────────────
  VCC            3.3V                    3.3V ONLY. 5V will damage sensor.
  GND            GND
  CS             D10                     Chip select (active LOW)
  SCL            D13                     SPI clock
  SDA            D11                     MOSI (controller → sensor)
  SDO            D12                     MISO (sensor → controller)

  SPI MODE: ADXL375 uses Mode 3 (CPOL=1, CPHA=1). Set correctly in firmware.
  If DEVID returns 0xFF instead of 0xE5, check CS pin and SPI mode first.

==============================================================================
BMI270 — NO WIRING REQUIRED
==============================================================================

  The BMI270 is soldered directly onto the Nano 33 BLE Rev2 PCB.
  It is accessed via the Arduino_BMI270_BMM150 library.
  No external connections needed.

  Library install:
    Arduino IDE → Tools → Manage Libraries → search "Arduino_BMI270_BMM150"

==============================================================================
MPU-9250 — NOT CONNECTED IN THIS REVISION
==============================================================================

  The MPU-9250 is awaiting header soldering.
  Once headers are soldered, switch back to Revision 2 firmware and
  follow the WIRING_REFERENCE from that revision for MPU-9250 connections.

==============================================================================
KNOWN LIMITATION OF THIS REVISION
==============================================================================

  The BMI270 gyroscope sample rate is ~100 Hz (library default), lower than
  the MPU-9250's configurable 500 Hz. This means:

  - The firmware loop runs at 500 Hz but gyro data only updates at ~100 Hz.
    Between gyro updates, the firmware reuses the last filtered value.
    This is handled gracefully via the IMU.gyroscopeAvailable() check.

  - Rotational acceleration (computed by differentiating gyro data) will
    have lower temporal resolution than with the MPU-9250.

  - For bench testing and pipeline validation, this is perfectly adequate.
    For final impact characterisation, switch to the MPU-9250.
